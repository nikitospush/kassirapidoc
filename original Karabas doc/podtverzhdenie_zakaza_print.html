Подтверждение заказа

 
 
 
  Команда подтверждает указанный заказ, изменяя его статус . Используется для резервирования, оплаты и отмены заказа.
 
ConfirmOrderCommand.cmd
Параметры
__auth
Ключ авторизации
Обязательный параметр
login
Логин учетной записиWEB-сайта
Обязательный параметр
password
Пароль учетной записиWEB-сайта
Обязательный параметр
format
По умолчанию JSON, или XML если передать format=xml
Формат выдачи данных
order
ID заказа
Обязательный параметр
deliveryType
Тип доставки
 
paymentType
Тип оплаты
 
client_id
ИД клиента
Если пусто, проверяется следующий параметр
client_card
Код карты клиента
Поиск клиента по коду карты. Если не найден, то следующий параметр
email
Емейл клиента
Поиск клиента по email. Если не найден – авторегистрация нового клиента
client_name
Имя клиента
 
address
Адрес доставки
 
comment
Комментарий клиента
 
cancelComment
Комментарий-причина отмены заказа
Нужен только при вызове с status = 2
status
Новый статус заказа
0 - Подтвердить резерв
1 - Подтвердить оплату
2 - Отменить заказ
3 – Отправить в доставку
deliveryDate
Конечная дата доставки
Используется только с status = 3
Устанавливает срок жизни заказа, отличный от стандартного. Если меньше значения по умолчанию, используется значение по умолчанию.
Ответ
order
ИД заказа
 
status
Новый статус заказа
0 - резерв
1 - оплачен
2 - отменен
3 – в доставке (ожидает денег)
amount
Сумма заказа
 
expires
Дата истечения заказа
Возвращается только в случае, если заказ находится в состоянии резерва.
tickets
Массив билетов
Возвращается при оплате заказа. Ниже следуют атрибуты объектов массива
id
Идентификатор билета
 
deliveryType
Тип доставки
TICKET_OFFICE - забрать в кассе
COURIER - доставка курьером
ETICKET - электронный билет
EMPTY - не задан
paymentType
Тип оплаты
CASH_TICKET - оплата на кассе
CASH_COURIER - оплата курьеру
LIQPAY - оплата LiqPay
BANK_PAY- банковский перевод
WEBMONEY - оплата WebMoney
WEBMONEY_TERMINAL - платежный терминал
PLATON - банковская карта
PORTMONE - оплата Portmone
EMPTY - не задан
barcode
Штрих-код
 
price
Цена
 
event
Мероприятие
 
Zone
Зона
 
Row
Ряд
 
Place
Место
 
eticket_url
ссылка на ПДФ файл электронного билета (если разрешено)
 
Важно! Штрих код будет валидный только после перевода заказа в статус 1 (оплачено). До перевода в данный статус,
штрих код может отсутствовать или изменится.
