Обновление схемы зала

 
 
  Поскольку состояние зала меняется в режиме реального времени (происходит продажа, резервирование мест), для отображения актуальной картины используется механизм Timestamp-ов.
  Для отрисовки изменений необходимо вызывать команду UpdateFlashEventDataCommand через определенные промежутки времени, в каждый последующий вызов передавать timestamp из предыдущего.
 
UpdateFlashEventDataCommand.cmd
Параметры
__auth
Ключ авторизации
Обязательный параметр
event
ИД мероприятия
Обязательный параметр
ts
Timestamp
Для первого вызова должен быть = 1, вернет весь зал
Ответ
Тег update
ts
значение Timestamp на сервере (сохранить и передать в следующий вызов)
ReserveColor
 
paidColor
цвет оплаченных мест
Тег p
Описание места
Id
идентификатор места
Status
Текущий статус места
0 - свободно
1 - продано
2 - текущий заказ
3 - не используется
4 - в резерве
Quote
квота, означающая кол-во доступных билетов на данное место. 0-недоступно, 1-обычное посадочное место, >1-фан-зона. Например, для кресла в театре Quote=1, для фан-зоны в клубе Quote=50
Free
кол-во мест, доступных к продаже
Price
цена
 
Пример XML-ответа:
<?xml version="1.0" encoding="utf-8"?>
